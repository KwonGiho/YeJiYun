QandA UX Redesign

UX Redesign

Date
July-August, 2016
Platform
Android App
Tasks
User data analysis
Persona
User flow
Usability test
UI planning
Tools
Adobe Illustrator
HTML
CSS
JavaScript

Summary

Math education service via sending photo. Students upload problem photo and they will get answer from teacher via photo. For their business goal that build recommendation system for student grade and their own progress, it is important to save users problem as dataset.

Problem


- User flow got complicated which makes users confuse to find way to get function
- Users want to delete their data when they finish own request that is quite critical issue to QandA recommendation system
- Main page is not organized based on user behavior

Process


1. User data analysis (analytics and users service center request)

To make research question clear and specific, figure out data which pages are popular and which contents users never pay attention to via google analytics. Data revealed study contents function browsed only 18%(5573) and other users’ request list is 3%(907).

1-1. Service center request

The biggest challenge is asking delete users request data. They want to other users do not know that they ask question. It is almost 30% of complaints.

2. Persona Redefinition

Initial persona was just middle and high school student. Clients thought real customer who pay for this app was parents. Based on their user profile data and complain center review, persona redefinition is prior challenge.

3.User flow reorganize

User flow was reorganized to be consistent in and out process. And scrape and library section consolidated previously duplicated across a service to support intuitive usage based on user’s mental model.

Usability test and Design

Lesson Learned

- Understanding clear persona is critical point since user might be totally different that what founder expect.
- Make simple flow as user’s mental model can help use intuitively.

Dungenium

Service & UI Planning

Date
January - February, 2016
Platform
Mobile Unity
UI planning
Game User research
Information architecture

Dungeonium is mobile defense social network game. Before my team was really impressed about Supercell, Clash of Clan. The success of SNG game is incredible powerful.

Summary

Information would be more valuable when people understand, organize and develop own their way not just accumulate. Tagit will support this.

Problem


Through user survey, we found what people feel inconvenient.
1. Many users shared for oneself or save link for future reference or reading later. However, it is a real hassle to find information without search system and only use scroll down.
2. Most users do not see again their saving contents so that makes information useless.

Process


1. Competitive Analysis

These service are not appropriate to user’s mental model ‘split-communicate-demand-achieve’ since it is not only for the accounting.

2. Affinity Test

- User flow should be aligned to mental model - The standards of calculation based on not event but official funds for easy to manage

3. Information Architecture

- User flow should be aligned to mental model - The standards of calculation based on not event but official funds for easy to manage

Solution

1. Tagit allows to search easily based on users’ tag and it increase credibility for usage

User can select specific contents in saved and shared lists on facebook and add their thinking to promote value.
